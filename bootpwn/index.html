Description

Secure boot is fundamental for assuring the authenticity of the Trusted Code Base (TCB) of secure devices. Recent attacks on Secure Boot implemented by a wide variety of devices (e.g. game consoles, mobile phones, you name it) are a clear indication vulnerabilities are widespread.

Are you ready to experience breaking Secure Boot using attacks beyond just software exploitation? Interested in identifying Secure Boot vulnerabilities and experiencing how they can be exploited using both software and hardware attacks?

Then, this the THE experience for you!

This is BootPwn, a gamified experience that is exercise driven and where it's unlikely you will get bored. You will take the attacker's seat in order to identify Secure Boot vulnerabilities applicable to real-world devices.

Using an emulated target, based on publicly available code bases and attacks, you will be challenged with interesting exercises at a rapid rate. The emulated target implements the ARMv8 (AArch64) architecture, but, at the same time, many of the exercises are architecture independent.

Please don't worry if your software reversing or exploitation skills are rusty. You do not need be a software security expert nor does this experience aims to make you one. Moreover, hardware attacks, like fault injection, are a relevant threat for Secure Boot implementations. These attacks will be discussed and simulated on the emulated target where possible.

Audience

  • Anyone with an interest in breaking Secure Boot on secure devices
  • Security enthusiasts with an interest in embedded device security
  • Designers of Secure Boot interested in an offensive perspective

    • Design information
